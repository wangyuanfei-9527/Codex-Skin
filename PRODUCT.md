# Product

## Purpose

Codex Skin Studio is a portable Windows theme-creation atelier. It converts a small image moodboard into a coherent Codex skin through visible, inspectable stages. Skin and pet creation remain separate until both can meet the same quality bar independently.

## Experience principles

1. Extract before designing. Show what subject, identity, traits, motifs, and risks were found.
2. Plan before generating. Produce the complete asset prompt pack before invoking image generation.
3. Generate before compiling. Never disguise a raw reference crop as a finished theme asset.
4. Preview before applying. Display the generated hero, generated icons, palette, copy, and native Codex layout together.
5. Fail closed. Missing, truncated, irrelevant, or invalid assets stop the workflow; they never trigger a fallback application.
6. Keep the change reversible. Only verified renderer DOM is restyled, and restore stays one click away.
7. Keep creative history. Every completed theme remains available locally for preview and switching until the user deliberately deletes it; the theme currently applied to Codex is always protected.

## Visual direction

The product should feel like a focused theme studio, not a Windows settings utility or generic AI dashboard. The composition stage is the center of gravity. Controls use restrained surfaces and few outlines; generated assets carry the theme identity.

## Acceptance

- Explicitly requested fictional characters remain recognizable.
- User-facing copy follows the user's language.
- Hero art has usable copy-safe space and contains no source text or fake controls.
- Four theme icons share one visual language and remain readable at small size.
- Sidebar, header, hero, cards, project selector, composer, and task surface read as one theme system.
- Native top menus and unrelated controls retain official geometry without injected outlines.
